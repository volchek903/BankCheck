# Telegram Finance Bot

Телеграм-бот на Python 3.11+, который по расписанию отправляет в личные сообщения подборки по банкам Беларуси: курсы валют, вклады, кредиты, лизинг и краткую сводку.

## Что умеет

- Отправляет отчеты каждый день в `07:00`, `12:00` и `18:00` по `Europe/Minsk`.
- Поддерживает ручной запуск отчетов через команды.
- Разделяет источники данных на отдельные провайдеры.
- Сохраняет предыдущее состояние в `app/storage/state.json` и показывает заметные изменения.
- Не падает целиком, если один источник или раздел временно недоступен.
- Ограничивает доступ одним пользователем из `.env`.

## Установка

1. Создайте виртуальное окружение:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Скопируйте пример окружения:

```bash
cp .env.example .env
```

4. Заполните `.env`.

## Как создать бота через BotFather

1. Откройте `@BotFather`.
2. Выполните `/newbot`.
3. Укажите имя и username бота.
4. Скопируйте токен и вставьте его в `BOT_TOKEN`.

## Как узнать свой Telegram ID

- Самый быстрый способ: написать любому боту, который показывает user id, например `@userinfobot`.
- Полученное число вставьте в `TELEGRAM_USER_ID`.

## Запуск локально

Обычный режим, в котором бот отвечает на команды и сам шлет отчеты по расписанию:

```bash
python -m app.main
```

Одноразовый запуск без ожидания расписания:

```bash
python -m app.main --once
```

Это удобно как тестовый режим и как ручной прогон после деплоя.

## Запуск в Docker

1. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN` и `TELEGRAM_USER_ID`.

2. Соберите и запустите контейнер:

```bash
docker compose up -d --build
```

3. Посмотрите логи:

```bash
docker compose logs -f bankcheck-bot
```

Одноразовая отправка отчетов через Docker:

```bash
docker compose run --rm bankcheck-bot python -m app.main --once
```

Остановить контейнер:

```bash
docker compose down
```

В Docker состояние бота хранится в volume `bankcheck-data` по пути `/data/state.json`, поэтому оно переживает пересоздание контейнера.

## Команды

- `/start` — проверить, что бот работает
- `/now` — вручную отправить все включенные отчеты
- `/currency` — только курсы валют
- `/deposits` — только вклады
- `/credits` — только кредиты
- `/leasing` — только лизинг
- `/summary` — краткая сводка
- `/help` — список команд

Если команду отправляет не тот пользователь, чей `TELEGRAM_USER_ID` указан в `.env`, бот отвечает:

```text
⛔ Доступ запрещен
```

## Как работает расписание

- Встроенный `APScheduler` запускает отчеты в `07:00`, `12:00` и `18:00`.
- Часовой пояс берется из `TIMEZONE`.
- Для нормальной работы расписания процесс должен быть постоянно запущен.

## Источник данных

Бот использует несколько рабочих источников:

- курсы валют: `select.by`
- вклады: `banki24.by`
- потребительские и автокредиты: `banki24.by`
- кредиты на недвижимость: `belapb.by`
- лизинг: `agroleasing.by`, `asbleasing.by`, `pal.by`

Что важно:

- часть источников не имеет официального API, поэтому используется аккуратный HTML-парсинг
- сайты могут менять разметку или временно ограничивать запросы
- в этом случае бот не падает полностью: проблемный раздел или подкатегория придет как временно недоступная, а остальные отчеты все равно отправятся

## Что именно отправляет бот

### 1. Курсы валют

Для `USD`, `EUR` и `RUB` бот ищет:

- лучший курс покупки валюты клиентом
- лучший курс продажи валюты клиентом

Логика направлений:

- если вы покупаете валюту у банка, выбирается минимальный курс продажи банка
- если вы продаете валюту банку, выбирается максимальный курс покупки банка

Дополнительно:

- сохраняется прошлый лидер и прошлый курс
- при изменении больше `0.5%` бот добавляет предупреждение

### 2. Вклады

Бот собирает топ предложений:

- отзывные
- безотзывные
- по возможности отдельно по `BYN`, `USD`, `EUR`, `RUB`

Также добавляются:

- лучшие вклады на короткий срок
- лучшие вклады на длинный срок
- уведомления о смене лидера

### 3. Кредиты

Бот ищет лучшие ставки по:

- потребительским кредитам
- кредитам на недвижимость
- автокредитам

Если на странице есть дополнительные условия, они тоже попадают в сообщение.

### 4. Лизинг

Бот собирает топ-3 предложений.

Если по предложению есть явная ставка, бот сортирует по ней. Если явной ставки нет, он использует доступные параметры страницы:

- аванс
- ежемесячный платеж
- срок

### 5. Краткая сводка

Сводка приходит отдельным сообщением и сжимает день до нескольких строк:

- где выгоднее купить и продать валюту
- какие вклады лидируют на короткий и длинный срок
- какие кредиты и лизинг сейчас наверху списка
- какие лидеры сменились

## Запуск на сервере

Рекомендуемый вариант: `systemd`, потому что боту нужен постоянный процесс для polling и встроенного scheduler.

### Пример systemd unit

```ini
[Unit]
Description=Telegram Finance Bot
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/telegram-finance-bot
ExecStart=/opt/telegram-finance-bot/.venv/bin/python -m app.main
Restart=always
RestartSec=5
User=financebot
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

После этого:

```bash
sudo systemctl daemon-reload
sudo systemctl enable telegram-finance-bot
sudo systemctl start telegram-finance-bot
sudo systemctl status telegram-finance-bot
```

## Если нужен cron

Если вы не хотите держать бот постоянно запущенным, можно использовать одноразовый режим `--once`.

Тогда cron просто запускает отправку в нужные часы:

```cron
CRON_TZ=Europe/Minsk
0 7,12,18 * * * cd /opt/telegram-finance-bot && /opt/telegram-finance-bot/.venv/bin/python -m app.main --once >> /var/log/telegram-finance-bot.log 2>&1
```

Этот вариант не нужен вместе со встроенным scheduler. Обычно выбирают что-то одно:

- либо постоянный процесс + `APScheduler`
- либо `cron` + `--once`

## Логи

Используется стандартный `logging`.

В логах видны:

- запуск приложения
- запуск расписания
- попытки отправки отчетов
- ошибки парсинга по отдельным разделам

## Ограничения и замечания

- часть агрегаторов может отдавать смешанные данные, поэтому для валютных вкладов бот дополнительно фильтрует предложения по названию продукта
- источники не гарантируют стабильный HTML и могут поменять структуру страниц
- В лизинге не на всех карточках публично указана процентная ставка.
- Dockerfile сознательно не добавлялся, чтобы проект оставался минимальным и без лишних артефактов.
